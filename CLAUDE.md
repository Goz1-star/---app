# 无名小卒工坊 App 产品文档

## 1. 项目名称与定位

### 1.1 项目名称

**无名小卒工坊**

### 1.2 产品定位

无名小卒工坊是一款面向软件技术专业学校工作室场景的成员学习与管理 App，帮助工作室室长或管理员完成成员管理、课程/活动组织、公告通知、打卡记录、积分激励、任务布置、资料沉淀、代码/文件提交管理、榜单对比和阶段性测试。

产品重点不是做复杂的社交平台或商业系统，而是服务工作室日常运营，让管理员能清楚掌握普通室员的学习、参与、成长和表现情况，也让普通室员能明确知道自己要做什么、学什么、完成了什么、排名如何。

### 1.3 产品目标

- 帮助管理员统一管理普通室员（学员）信息。
- 帮助管理员发布课程/活动、公告、任务、资料、代码作业和测试。
- 帮助普通室员完成打卡、任务提交、代码/文件上传、资料学习和阶段性测试。
- 通过积分奖惩和对比榜单形成成长激励。
- 通过清晰的权限划分，避免普通室员访问管理员功能。

## 2. 用户角色定义

### 2.1 管理员

管理员指工作室室长或被室长授权的管理者，负责维护工作室内容、管理普通室员、布置任务、发布公告、配置积分奖惩，管理代码/文件提交，并查看普通室员的打卡、任务、代码作业、测试和榜单表现。

### 2.2 普通室员（学员）

普通室员和学员在本产品中按同一种用户处理。普通室员是参与工作室学习、活动、任务、打卡、代码/文件提交和测试的成员。

普通室员主要负责：

- 查看公告和课程/活动信息。
- 完成拍照打卡或时长打卡。
- 接收并提交管理员布置的任务。
- 上传代码、压缩包、文档、图片等作业文件。
- 查看资料仓库中的学习资料。
- 查看个人积分、榜单排名和阶段性测试结果。

### 3.1 权限原则

- 管理员和普通室员使用不同登录入口。
- 登录成功后，系统根据角色进入不同首页。
- 普通室员不能访问管理员后台功能。
- 普通室员不能新增、修改、删除其他成员信息。
- 普通室员不能发布公告、调整积分、上传公共资料或创建测试。
- 管理员可查看普通室员的学习、打卡、任务、代码/文件提交、积分、测试等运营数据。

## 4. 登录与访问流程

### 4.1 登录入口

App 需要区分两个登录页面：

1. **管理员登录页面**
   - 面向管理员使用。
   - 登录后进入管理员端首页/数据概览。

2. **学员登录页面**
   - 面向普通室员（学员）使用。
   - 登录后进入普通室员端首页。

### 4.2 登录方式

管理员和普通室员登录页都支持以下方式：

- 账号密码登录
- 手机号验证码登录

首版手机号验证码先使用**模拟验证码**，用于完成登录流程验证；后续如需要正式上线，再接入真实短信服务。

### 4.3 管理员登录流程

1. 用户进入管理员登录页。
2. 选择账号密码或手机号验证码登录。
3. 系统校验账号是否为管理员身份。
4. 校验成功后进入管理员端首页。
5. 如果账号不是管理员，应提示无管理员权限，禁止进入管理员端。

### 4.4 普通室员登录流程

1. 用户进入学员登录页。
2. 选择账号密码或手机号验证码登录。
3. 系统校验账号是否为普通室员身份。
4. 校验成功后进入普通室员端首页。
5. 如果账号不是普通室员，应提示账号身份不匹配。

## 5. 首版核心功能说明

### 5.1 成员管理

管理员可管理普通室员信息。

首版能力：

- 新增普通室员。
- 编辑普通室员资料。
- 查看普通室员列表。
- 查看普通室员详情。
- 禁用或恢复普通室员账号。
- 查看普通室员的积分、打卡、任务、测试等基础表现。

普通室员能力：

- 查看自己的个人资料。
- 查看自己的学习与成长记录。

### 5.2 课程/活动管理

管理员可发布工作室课程或活动，普通室员可查看和参与。

首版能力：

- 管理员创建课程/活动。
- 管理员编辑课程/活动名称、时间、说明、参与对象。
- 管理员发布或下架课程/活动。
- 课程/活动需要报名审核，普通室员报名后由管理员审核通过后才能正式参与。
- 普通室员查看课程/活动列表。
- 普通室员查看课程/活动详情。

### 5.3 公告通知

管理员可发布通知，普通室员可查看。

首版能力：

- 管理员发布公告。
- 管理员编辑或删除公告。
- 普通室员查看公告列表。
- 普通室员查看公告详情。
- 普通室员可看到重要通知或最新通知提醒。

### 5.4 简易拍照打卡 / 时长打卡

用于记录普通室员的参与和学习情况。

首版支持两种打卡形式：

1. **拍照打卡**
   - 普通室员上传照片作为打卡凭证。
   - 可填写打卡说明。
   - 管理员可查看打卡记录。

2. **时长打卡**
   - 普通室员记录学习或参与时长。
   - 可填写学习内容或备注。
   - 管理员可查看时长统计。

### 5.5 积分奖惩

积分用于激励普通室员持续参与和成长。

积分来源可以包括：

- 完成打卡获得积分。
- 完成任务获得积分。
- 提交代码/文件作业并通过评价获得积分。
- 阶段性小测试获得积分。
- 管理员手动奖励积分。

积分扣减可以包括：

- 管理员手动扣减。
- 未完成任务或违反规则时扣减（具体规则待确认）。

首版能力：

- 管理员查看成员积分。
- 管理员手动增加或扣减积分。
- 普通室员查看自己的积分总数和积分明细。
- 代码/文件作业可根据完成质量获得积分奖励。
- 积分可用于榜单排名。

### 5.6 布置任务

管理员可向普通室员布置任务，普通室员提交完成结果。

首版能力：

- 管理员创建任务。
- 管理员设置任务标题、说明、截止时间、参与对象。
- 普通室员查看待完成任务。
- 普通室员提交任务内容、图片、代码文件、压缩包、文档或附件说明。
- 管理员查看任务提交情况。
- 管理员可给出反馈或评分。

### 5.7 代码/文件提交

因为无名小卒工坊面向软件技术专业的学校工作室，首版需要支持成员上传代码和学习文件，方便管理员检查作业、项目阶段成果和练习记录。

首版支持的提交内容：

- 代码文件，例如 `.html`、`.css`、`.js`、`.ts`、`.vue`、`.java`、`.py`、`.cpp` 等。
- 项目压缩包，例如 `.zip`、`.rar`、`.7z`。
- 文档文件，例如 `.md`、`.txt`、`.docx`、`.pdf`。
- 图片文件，例如项目截图、运行结果截图、设计图。
- 外部代码仓库链接，例如 GitHub 仓库地址。

首版能力：

- 普通室员在任务中上传代码/文件或填写仓库链接。
- 普通室员可查看自己的提交记录和管理员反馈。
- 管理员可查看、下载、在线预览成员提交的代码文件。
- 管理员可对代码/文件提交进行评价、打分、退回修改或给予积分奖励。
- 系统记录提交时间、提交人、所属任务、文件类型、反馈状态。
- 支持将代码提交写入或关联到 GitHub，其他代码托管平台首版暂不支持。

### 5.8 资料仓库

资料仓库用于沉淀课程资料、学习文档、图片、链接等内容。

首版能力：

- 管理员上传或创建资料。
- 管理员对资料进行分类。
- 管理员编辑或删除资料。
- 普通室员查看资料列表。
- 普通室员查看、下载或学习资料。

### 5.9 对比榜单

榜单用于展示普通室员之间的成长对比，形成正向激励。

榜单维度可包括：

- 积分榜。
- 打卡榜。
- 任务完成榜。
- 代码/文件提交质量榜。
- 阶段性小测试成绩榜。

榜单首版为**全员可见**，普通室员可以查看完整榜单及自己的排名。

首版能力：

- 管理员查看全员榜单。
- 普通室员查看榜单和自己的排名。
- 榜单可按时间周期筛选，例如本周、本月、全部。

### 5.10 阶段性小测试

管理员可创建阶段性测试，用于检验普通室员学习成果。

首版能力：

- 管理员创建测试。
- 管理员维护题目。
- 普通室员参加测试。
- 系统记录测试成绩。
- 阶段性小测试支持单选、多选、判断、简答题。
- 管理员查看测试结果。
- 普通室员查看自己的测试结果。

### 5.11 数据导出

管理员端需要支持数据导出，方便室长进行工作室管理、成果统计、阶段复盘和资料备份。

首版可导出的数据包括：

- 成员名单。
- 打卡记录。
- 积分记录。
- 任务提交记录。
- 代码/文件提交记录。
- 课程/活动报名与审核记录。
- 阶段性小测试成绩。

导出格式首版只需要支持 Excel。

## 7. 业务规则

### 7.1 账号与角色规则

- 管理员和普通室员是不同角色。
- 普通室员和学员在本产品中为同一类用户。
- 管理员登录页只允许管理员身份进入。
- 学员登录页只允许普通室员身份进入。
- 同一账号允许同时拥有管理员和普通室员身份。
- 当同一账号拥有多个角色时，应根据登录入口决定进入管理员端或普通室员端。
- 首版角色只保留室长/管理员和普通室员（学员），不增加老师、导师、助教等第三种角色。

### 7.2 权限规则

- 普通室员只能操作自己的学习、打卡、任务、代码/文件提交、测试相关内容。
- 管理员可以管理全部普通室员的运营数据。
- 普通室员不能访问管理员端页面。
- 所有涉及成员管理、内容发布、积分奖惩、资料管理、代码/文件提交评价、测试管理的功能都需要管理员权限。

### 7.3 积分规则

- 积分可以由打卡、任务、代码/文件作业、小测试和管理员手动奖惩产生。
- 每一次积分变化都应保留明细记录。
- 榜单可优先基于积分进行展示。
- 积分规则应支持后续扩展，例如不同任务给不同积分。

### 7.4 打卡规则

- 普通室员可进行拍照打卡或时长打卡。
- 打卡记录归属于提交人。
- 打卡不需要地理位置定位。
- 打卡不需要管理员审核。
- 允许普通室员补卡。
- 普通室员补卡时可填写补卡原因，但不强制填写。
- 管理员可查看和统计打卡记录。

### 7.5 资料规则

- 资料仓库由管理员维护。
- 普通室员主要进行查看、下载和学习。
- 资料可按分类管理，方便查找。

### 7.6 代码/文件提交规则

- 代码/文件提交通常归属于某个任务或课程/活动。
- 普通室员只能查看和管理自己的提交记录。
- 管理员可以查看、下载、评价全部普通室员提交的代码/文件。
- 每次提交应记录提交人、提交时间、文件名称、文件类型、所属任务和反馈状态。
- 代码文件上传限制为单文件最大 200MB。
- 其他文件类型首版不设置特殊大小限制，但仍应受系统整体存储能力约束。
- 对代码文件应支持在线预览，方便管理员快速查看作业内容。
- 对代码文件应以学习检查为主，首版不要求在线编译或在线运行代码。
- GitHub 作为首版唯一支持的外部代码仓库平台。
- GitHub 写入方式允许管理员或普通室员在提交时选择，不强制所有代码提交都写入 GitHub。
- 外部 GitHub 仓库链接可作为补充提交方式，方便软件技术专业成员提交完整项目。

### 7.7 课程/活动报名审核规则

- 课程/活动需要报名审核。
- 普通室员提交报名申请后，状态为待审核。
- 管理员审核通过后，普通室员才算正式参与课程/活动。
- 管理员可以拒绝报名，并可填写拒绝原因。
- 普通室员可查看自己的报名审核状态。

### 7.8 数据导出规则

- 数据导出功能仅管理员可用。
- Web 管理后台只给管理员使用，普通室员不使用 Web 管理后台。
- 普通室员不可导出全员数据。
- 导出数据应至少支持成员名单、打卡记录、积分记录、任务提交、代码/文件提交、课程/活动报名审核记录和小测试成绩。
- 导出时应支持按时间范围、成员、课程/活动或任务进行筛选。
- 首版数据导出只需要支持 Excel 格式。

## 8. 非目标 / 暂不实现范围

首版暂不设计以下能力，除非后续明确提出：

- 支付、商城、订单、复杂财务系统。
- 多工坊、多门店、多租户 SaaS 架构。
- 即时聊天、朋友圈式社交、直播。
- 复杂 AI 功能。
- 在线代码编译、在线代码运行、自动判题系统。
- 复杂审批流系统。
- 复杂组织架构和多级管理员体系。

9. 代码文件上传限制为单文件最大 200MB。
10. 其他文件类型首版不设置特殊大小限制，但仍受系统整体存储能力约束。
11. 首版需要写入或关联 GitHub，其他代码托管平台暂不支持。
12. 需要在线预览代码文件。
13. 需要支持数据导出。
14. 需要 Web 管理后台。
15. 同一账号允许同时拥有管理员和普通室员身份。
16. 普通室员补卡原因不强制填写。
17. GitHub 写入可以选择，不强制所有代码提交都写入 GitHub。
18. 数据导出只需要 Excel 格式。
19. Web 管理后台只给管理员使用。

## 10. 待确认问题

当前关键产品规则已基本确认。后续进入原型或开发前，可继续细化页面交互、字段设计、积分计算方式和 GitHub 具体接入方式。

## 11. 后续实现建议

后续如果开始进入 App 设计和开发，建议按以下顺序推进：

1. 先确定技术形态：管理员端需要 Web 管理后台，且 Web 管理后台只给管理员使用；普通室员端可根据后续需要选择 Web App、移动 App 或小程序。
2. 先实现账号、角色、登录和权限隔离。
3. 再实现管理员端基础后台：成员、公告、课程/活动。
4. 再实现普通室员端基础功能：查看公告、课程/活动、个人中心。
5. 再实现打卡、积分、任务、代码/文件上传、资料仓库、榜单和小测试。
6. 最后补充统计、筛选、导出、通知等增强能力。
